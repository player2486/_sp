P0 編譯器（由 Niklaus Wirth 設計）的函數呼叫機制走的是**「極簡堆疊路徑」**，核心原理如下：

1. 建立環境 (Setup)
當程式偵測到呼叫時，會依序將參數壓入堆疊。如果是巢狀函數，還會壓入一個靜態鏈 (Static Link)，讓函數能找到外層區域的變數。

2. 保存與跳轉 (Call)
執行跳轉指令（如 BSR），同時將返回地址（下一條要執行的指令位置）存入堆疊，確保執行完後回得來。

3. 分配空間 (Allocation)
進入函數後，編譯器會移動堆疊指標（SP），為該函數的區域變數預留記憶體空間。

4. 清理與返回 (Return)
函數結束時，執行反向操作：

釋放空間：將堆疊指標移回，丟棄區域變數。

恢復地址：從堆疊取出返回地址，跳回原呼叫處。

傳回值：通常放在約定的暫存器（如 R0）中。
